## What we are building

Promptbox turns a folder of Markdown notes into a searchable, filterable library of AI prompts inside Obsidian. Each prompt is a plain note with YAML frontmatter, so the library stays readable and portable even without the plugin installed. Users find prompts through combinable filters and full-text search, and reuse them via clipboard copy with guided variable filling. Everything stays local: no server, no network calls, no telemetry.

Full requirements: `docs/spec.md` (MVP) and `docs/spec-community.md` (Phase 2).

## Why

Prompt collections scattered across notes are hard to find and reuse. A structured, local-first library inside the tool people already write in (Obsidian) removes that friction without locking content into a proprietary format.

## Scope

**In scope (v1 / MVP, Tiers 0-7):**
- Prompts as portable Markdown notes with structured metadata (ADR-0001)
- Library view with combinable filters, full-text search, sort
- Full CRUD through the plugin
- Reuse engine: clipboard copy with guided variable filling, quick picker
- JSON import/export
- Hardening, performance, Obsidian store submission

**Out of scope (for now, per spec.md §1.2):**
- NG-1 community/shared library features (Phase 2, design only)
- NG-2 prompt execution against AI providers, API keys, chat UI
- NG-3 version history or diffs
- NG-4 sync/backup beyond JSON export
- NG-5 encryption or access control
- NG-6 UI localization (English only)

## Architecture decisions

- ADR-0001 (accepted): storage as plain Markdown + YAML frontmatter
- ADR-0002 (accepted): UI on native Obsidian components, vanilla TypeScript, esbuild
- ADR-0003 (proposed): community library via GitHub-based catalog with thin submission bridge

See `docs/adr/README.md`.
